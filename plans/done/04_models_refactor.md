## Detailed Changes

### DELETE: `agent_reports.py`

**Why:** Every class in this file is only used by the agents being deleted:

- `TechnicalReport` → used by `TechnicalAgent` and `Aggregator`
- `FundamentalReport` → used by `FundamentalAgent` and `Aggregator`
- `SentimentReport` → used by `SentimentAgent` and `Aggregator`
- `RiskReport` + `ScenarioCase` → used by `RiskAgent` and `Aggregator`

All of these flow into the `Aggregator` → `RulesEngine` path, which is being deleted.

The `_backfill_from_reasoning()` and `_extract_dollar_levels()` hacks are evidence of how often the LLMs fail to produce these schemas — another reason to eliminate them.

### DELETE: `decision.py`

**Why:**

- `RuleEvaluation` is only used by `FinalDecision`
- `FinalDecision` is only used by `RulesEngine.evaluate()` and `PipelineService`
- Both are being deleted/refactored

The strategist produces its own trade decisions directly via tool calls (`place_buy`, `place_sell`), not via Pydantic models.

### SIMPLIFY: `dossier.py`

**What stays:**

- `QuantScorecard` — **Keep exactly as-is.** This is the quant engine output, used by the strategist, data distiller, and stored in DuckDB. It's clean and well-structured.

**What gets deleted:**

- `QAPair` — Only used by the RAG engine (Layer 3), which is being deleted
- `TickerDossier` fields change:
  - Remove `qa_pairs: list[QAPair]` (no more RAG)
  - Keep `executive_summary`, `bull_case`, `bear_case`, `conviction_score` — but these will now be generated by the strategist's reasoning, not a separate LLM call
  - Simplify: the `TickerDossier` becomes a thin wrapper around `QuantScorecard` + distilled text analysis

**New simplified `TickerDossier`:**

```python
class TickerDossier(BaseModel):
    """Analysis package for a single ticker — quant scores + distilled context."""
    
    ticker: str
    generated_at: datetime = Field(default_factory=datetime.now)
    
    # Layer 1: Pure math
    quant_scorecard: QuantScorecard
    
    # Distilled analysis (from DataDistiller — no LLM calls)
    price_action_analysis: str = ""
    fundamental_analysis: str = ""
    risk_analysis: str = ""
    sentiment_analysis: str = ""  # NEW: from moved sentiment formatting
    
    # Signal summary (one-liner from quant signals)
    signal_summary: str = ""
```

No more `bull_case`, `bear_case`, `conviction_score`, `key_catalysts` — those are the strategist's job to determine, not pre-computed.
